Game dev uncertainty is expensive. We run an 8-week kill-or-scale system based on Lean principles. Each cycle tests a game hypothesis via Minimum Viable Experiment (MVE) ‚Äì the simplest build to gather validated learning. We track Actionable Metrics (like D1 Retention, Conversion) against goals & gather player feedback. AI compresses the 4-week build phase, freeing the Measure & Learn phase for data analysis. Kill/Pivot/Persevere decisions are data-driven, not gut feelings. This systematically finds Product-Market Fit (PMF).

üß™ The MPS Innovation Engine üî¨

  • Test the Core Loop, Nothing Else

    Ideas hit player data in 8 weeks (4 Build, 4 Measure/Learn). We build only the MVE's Core Loop to test engagement (e.g., D1 Retention). Scope is ruthlessly minimized; non-essentials wait for data proof.

  • AI Accelerates Build, Humans Analyze

    AI speeds execution (code, assets, tagging). This focuses humans on critical thinking: sharp hypotheses, Customer Development, interpreting the 'why' behind KPIs.

  • Mandatory Post-Mortems Drive Learning

    Each 8-week cycle ends with a mandatory Post-Mortem. Killed MVEs yield negative data. Successes refine patterns, failures highlight risks. Validated Learning is codified to improve future cycles.

  • Data Dictates Decisions via KPI Matrix

    No sunk cost fallacy. Our KPI Decision Matrix sets clear thresholds for goals (D1 Retention, etc.) & feedback. Underperforming MVEs are killed or pivoted based on this data. Swiftly.

  • Scale Only Proven Loops

    MVEs showing strong PMF (hitting KPIs, positive feedback) trigger Persevere. Scaling is a separate process. The core R&D team starts the next 8-week MVE cycle, maximizing validation throughput.

  • Relentlessly Optimize the MVE Engine

    The 8-week process itself is continuously improved. Post-Mortem insights update our Playbook, Battle Plan, & tools. We eliminate operational WASTE hindering faster Validated Learning.
